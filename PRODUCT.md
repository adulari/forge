# Forge Product Context

## Register

Product UI. Forge is a local-first coding harness for developers working in a terminal.

## Purpose

Forge helps a developer understand, change, review, and ship a codebase through a direct chat
workflow. Its model mesh, workflows, agents, tools, and integrations add capability but must not
obscure the basic act of describing a task and reviewing the result.

## Interaction Principles

- Conversation is the default surface; advanced controls are progressive disclosure.
- Every feature remains searchable and reachable from one command center.
- Live work communicates a concise outcome first and exposes operational detail on demand.
- Commands that can change code or session state remain explicit and reviewable in the composer.
- Keyboard operation is first-class, with mouse support as a convenience.

## Personality

Calm, capable, and direct. Forge should feel like a focused engineering tool, never a dashboard
asking for attention.

## Accessibility

Use high-contrast text and non-color state cues. Animation communicates live state only and must
remain understandable when a terminal does not support color or animated glyphs.
